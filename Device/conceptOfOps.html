overview
the device sends changes for storage
device sends changes to html page (devlog.html)
devlog.html calls devlog.php
devlog.php via sr232 transfers data to phpReader.py
phpReader writes QML message to SOA
SOA directs the task of logging device changes

the user requests an average be created
user requests via userRequest.html for "Average" viz command picklist
user logs off

the user checks their status reports
user requests via userRequest.html for "Status" viz command picklist
if the final processing of the device changes average has been completed; the "average" report will be part of the report package.
Goals
Technical goal to prove:
1- that messages can be transfered between php and python without sockets
2- that an agent can do some work (log changes, create an average )
3- that an SOA task can be built from point diagrams
4- that an SOA agent can orchestrate inter-agent work to accomplish a task

Business goal to prove:
1- that a platform can be presented to the medical device community to:
----a) have some device enter information simply into a larger system.
----b) allow different analysis to be performed on the information provided.
----c) allow results to be published to online subscribers.
